## 🔑 5言語を貫く3つの軸

同じ概念でも、言語によって挙動が根本的に変わるポイントがこの3つです。読むときはここに注目すると差分がクリアになります。

**① 変数はオブジェクト本体か、番地（参照）か**
- Python / Java / TypeScript … 変数は**参照**（番地）。`b = a` はエイリアス（片方を変えると両方変わる）。
- C++ … デフォルトは**値**。`b = a` は**コピーで別実体**。エイリアスにはポインタ／参照が必要。
- Rust … `b = a` は**ムーブ**（所有権が移り、元の `a` は使えなくなる）。共有は借用 `&`。

**② メモリはいつ・誰が解放するか**
- Python … 参照カウントが0で**即座**に解放（＋循環用GC）。
- Java / TypeScript … **GC**が回収（タイミングは不定）。
- C++ … スコープ終端 or `delete` で**確定的**（RAII / スマートポインタ）。
- Rust … 所有権が尽きるスコープ終端で**確定的**（`Drop`、コンパイル時に決定）。

**③ 破棄時の後始末（デストラクタ）をどう書くか**
- C++ = `~ClassName()` / Rust = `Drop` トレイト … 明示的に書き、確定的に呼ばれる。
- Python = `__del__`（不確実）→ 資源は `with` / Java = `finalize`（非推奨）→ `try-with-resources` / TypeScript = デストラクタなし → `using`。
